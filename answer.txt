Why should you have different branches?
If we just have one main branch, it is very tough to switch version when there are too many changes.
Also, if there is a problem with master branch, we will not be able to continue development.

In a project how many remote repositories do you usually have?
You can have as many remote repos as you want, but there is only remote repo called origin.
